## 🎯 The Motivation (Edge AI Software Engineering)

In applied AI software engineering specifically for unmanned systems (UAVs, UxVs, and robotics), simply scaling up Transformers or chasing raw accuracy is highly dangerous. Deploying logic onto aerial devices requires an acute awareness of strict **SWaP constraints (Size, Weight, and Power)**. 

To safely map and deploy models onto edge processing units (like NVIDIA Jetson or custom ARM SoCs), we have to profile hardware-level trade-offs: latency, memory footprint, and MACs/FLOPs. 

This toolkit provides a highly reproducible, fully interactive pipeline to evaluate the most cutting-edge Vision Transformers and Convolutional networks—specifically edge-optimized architectures from **2024 and 2025**—under identically controlled constraints on real-world **EuroSAT (Drone/Satellite) image distributions**.

## 🚀 The Architectures (2025 SOTA Focus)

The registry natively pulls from **Hugging Face (`timm`)** and `torchvision`. It focuses deeply on extremely lightweight models evaluated specifically for their representation power on aerial geographical topology:

### Modern SOTA Edge (2023 - 2025)
* **MobileNetV4** *(Google, 2024)* — Universal hardware abstraction CNN.
* **RepViT** *(Tsinghua, 2024)* — Re-parameterized CNN-ViT hybrid built for speed.
* **EfficientViT** *(MIT, 2023)* — Linear attention ViT dominant on memory-bound UAV hardware.
* **FastViT-MCI4** *(Apple, 2025 Iteration)* — Hybrid topological design pushing absolute SOTA zero-shot latency constraints.

### Industry Standards & Baselines
* **ConvNeXt-Tiny** *(2022)* — Modernized pure-CNN design.
* **Swin-Tiny / MaxViT-Tiny** *(2021/2022)* — Foundational Transformer baselines.
* **MobileNetV3 / EfficientNet** *(2019)* — The ubiquitous edge deployments.

## ⚙️ Interactive Pipeline

Forget complex CLI arguments. This toolkit is designed to be purely interactive, providing an intuitive terminal user interface (TUI) for execution and visualization.

### 1. Execute the Benchmark
```bash
conda activate pipeline_c  # Or your chosen environment
python 01_run_benchmark.py
```
* **Interactive Prompts**: Choose your execution device (NVIDIA CUDA or CPU fallback) and select specific architectures from the dynamic registry.
* **Automated Data**: Automatically pulls and constructs the EuroSAT spatial imagery dataloader.
* **Telemetry Gathered**: Top-1/Top-5 Accuracy, Model Parameter Footprint, CPU/GPU Latency Profiles, and Batch Throughput.

### 2. Generate Pareto Visualizations
```bash
python 02_generate_visualizations.py
```
* Automatically reads the JSON telemetry payload generated by script `01`.
* Reconstructs publication-ready Pareto-front scatter plots (Accuracy vs. Latency) mapped by model parameter footprint.

## 🔬 Evaluation Rigor

* **Footprint**: Extracted directly using `ptflops`.
* **Latency**: Measured via `torch.cuda.Event` (for GPU) and `time.perf_counter` (for CPU) with strict 10-iteration warmups and calculated medians over 50 runs to eliminate OS-level jitter.
* **Fairness**: Identical spatial geometry transformations applied across all networks prior to evaluation to mimic real-time aerial sensor feed limitations.
