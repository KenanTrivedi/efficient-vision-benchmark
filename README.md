## 🎯 Motivation

In applied AI software engineering (e.g., UAVs, robotics, edge embedded systems), simply chasing raw accuracy is insufficient. Deployment requires an acute awareness of **trade-offs**: latency, memory footprint, and MACs/FLOPs. Published numbers often use disparate hardware and augmentation recipes, making direct comparisons impossible.

This toolkit provides a highly reproducible, fully interactive pipeline to evaluate the most cutting-edge Vision Transformers and Convolutional networks—specifically edge-optimized architectures from **2024 and 2025**—under identically controlled constraints on real-world distributions.

## 🚀 The Architectures (2025 SOTA Focus)

The registry natively pulls from **Hugging Face (`timm`)** and `torchvision`. It focuses deeply on extremely lightweight models designed for high-speed edge inference:

### Modern SOTA Edge (2023 - 2024)
* **MobileNetV4** *(Google, 2024)* — Universal hardware abstraction CNN.
* **RepViT** *(Tsinghua, 2024)* — Re-parameterized CNN-ViT hybrid built for speed.
* **EfficientViT** *(MIT, 2023)* — Linear attention ViT dominating memory-bound hardware.
* **FastViT** *(Apple, 2023)* — Hybrid topological design for hyper-fast latency.

### Industry Standards & Baselines
* **ConvNeXt-Tiny** *(2022)* — Modernized pure-CNN design.
* **Swin-Tiny / MaxViT-Tiny** *(2021/2022)* — Foundational Transformer baselines.
* **MobileNetV3 / EfficientNet** *(2019)* — The ubiquitous mobile deployments.
* **ResNet-50** *(2016)* — The perennial benchmark.

## ⚙️ Interactive Pipeline

Forget complex CLI arguments. This toolkit is designed to be purely interactive, providing an intuitive terminal user interface (TUI) for execution and visualization.

### 1. Execute the Benchmark
```bash
conda activate pipeline_c  # Or your chosen environment
python 01_run_benchmark.py
```
* **Interactive Prompts**: Choose your execution device (NVIDIA CUDA or CPU fallback) and select specific architectures from the dynamic registry.
* **Automated Data**: Automatically downloads and stages CIFAR-10.
* **Telemetry Gathered**: Top-1/Top-5 Accuracy, Model Parameter Footprint, CPU/GPU Latency Profiles, and Batch Throughput.

### 2. Generate Pareto Visualizations
```bash
python 02_generate_visualizations.py
```
* Automatically reads the JSON telemetry payload generated by script `01`.
* Reconstructs publication-ready Pareto-front scatter plots (Accuracy vs. Latency) mapped by model parameter size.

## 🔬 Evaluation Rigor

* **Footprint**: Extracted directly using `ptflops`.
* **Latency**: Measured via `torch.cuda.Event` (for GPU) and `time.perf_counter` (for CPU) with strict 10-iteration warmups and calculated medians over 50 runs to eliminate OS-level jitter.
* **Fairness**: Identical standard `ImageNet` preprocessing applied across all networks prior to evaluation.
